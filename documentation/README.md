# /documentation

SOPs, playbooks, standards, and the knowledge base. Internal process only —
never client work. Owned by the `documentation` agent.

## Rules

- Flat until it isn't. Add a subfolder only when one topic exceeds roughly five
  files.
- One source of truth per topic. A duplicate is a defect.
- A document that maps to no success criterion should not exist.
- Client work lives in `projects/{client-slug}/{YYYY}/`. Never here.

## Success criteria served

| Criterion | How |
|-----------|-----|
| Consistency | Quality does not depend on who did the work |
| Internal efficiency | Process is reused, not rediscovered |
| Scalability | Ramp time drops as volume rises |
